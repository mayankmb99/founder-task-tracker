# Pitch deck content

Slide 1: Problem and user

- Founders waste time manually extracting tasks and preparing for meetings.
- The product serves founders who need reliable follow-through and grounded meeting prep.

Slide 2: Insight and why now

- Founder workflows span email, calendar, tasks, and meeting context.
- AI is useful only when it is structured, grounded, and approved by the user.

Slide 3: Solution and demo

- Task extraction with Add / Edit / Dismiss.
- Preparation strategy generation for sales, investor, partnership, vendor, customer-call, startup-event, conference, and other interactions.

Slide 4: AI and technical architecture

- One shared OpenAI client.
- Responses API + Structured Outputs.
- Prompt modules, context builders, validation, telemetry, retries, and optional File Search.

Slide 5: Value and GTM

- Reduces manual triage.
- Improves meeting readiness.
- Keeps persistence server-side and human-controlled.

Slide 6: Roadmap, risks, and ask

- Enable preferred models once verified.
- Turn on document retrieval when vector store is configured.
- Improve eval coverage as new workflows are added.
- Ask: pilot users / feedback / distribution support.
